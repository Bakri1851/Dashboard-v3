\section{Introduction} \label{sec: intro}
\subsection{Problem}

Computer Labs are fast paced environments where many students are working simultaneously on different questions are their own pace.
Some students complete questions slower than others, and some may have prior knowledge.
In practice, it is difficult to provide timely, high-quality support to students, monitor their progress, and prioritise those who are struggling most so they do not fall behind. We aim to address these issues in this paper.

Our current computer labs are highly dependent on staff surveying the room or counting on students asking for help.
The current approach to computer labs has a range of limitations.
Students who hesitate to ask for help in a lab environment may not receive the support they need.
A student can be struggling significantly, but we may have no indicator of this struggle.
Additionally, knowing what students require more help is almost impossible with the current approach, even with staff surveying the lab it is hard to quickly judge who needs immediate help.
Furthermore, there is no structure to the allocation of lab assistants, without any indicator of who is struggling and allocating students smartly we do not provide help in the most efficient way possible.

These limitations become more severe the more class size increases.
With more students, it becomes harder to identify struggling students and prioritise them for help.

The central problem is addressed in this project is therefore how to provide timely, high-quality feedback to students in computer labs, using the data generated from student work.
The goal is not to replace human teaching but to enhance it.

\subsection{Proposed Solution}


In this project, we had addressed these issues and limitations with a real-time learning analytics dashboard we developed for use in computer lab sessions.
The system supports lab teachers and assistants by transforming live student data into interpretable metrics we can use quickly during labs.


The dashboard integrates data from an existing endpoint and processes it into a set of practical metrics.
The include measures derived from attempt number, time spent and average numbers of attempts to complete a question.
From these signals we compute indicators of student struggle and question difficulty, allowing staff to identify which students are struggling and which questions are causing the most difficulty.

The system aims to assist rather than replace human judgement.
The dashboard does not by default make its own decisions.
Instead it provides a clearer operational view of what is happening in the lab so instructors can allocate resources more effectively.
The system also allows for lab instructors to allocate lab assistants within the session, allowing them to be allocated through the system rather than informally.

The project has smart device integration, sending who to help to lab assistant's phones with recommendations on how to help students.
\subsection{Aims and Objectives}


The primary aim of this project was to design and implement a system that enables lab teachers and lab assistants to make more well-informed decisions during computer lab sessions.

The secondary aim is to extend our system to smart devices. Whilst the core system is a desktop-based dashboard, we can improve usability by sending notifications or messages to smart devices to enable easier communication with assistants.

The aims will be addressed through the following objectives:
\begin{enumerate}
    \item \textbf{Identify relevant literature and existing systems} related to learning analytics, real-time data visualisation, and AI in education, to identify existing limitations in current approaches to relevant areas
    \item \textbf{Formalise system requirements} by defining functional and non-functional requirements for the real-time computer lab support system
    \item \textbf{Design Data Model} that extracts relevant data provided into meaningful metrics suitable for real-time analysis
    \item \textbf{Design system architecture} to support live data ingestion, prioritisation and presentation while remaining extensible to smart devices
    \item \textbf{Develop a struggle and difficulty model} so that we can take the relevant values to produce metrics which will be used in showing the relevant information 
    \item \textbf{Implement live data analytics dashboard} for real-time data visualisation, prioritisation and intuitive interactivity
    \item \textbf{Evaluate approach} through functional and non-functional aspects to determine the feasibility and usefulness in a computer lab session
\end{enumerate}

Together, these objectives guided the project toward becoming a full-fledged EdTech application that aimed to improve learning outcomes by enabling more efficient and targeted support.

\subsection{Project Approach}

The project followed an incremental and implementation-focused approach.
Early work focused on understanding the educational and technical context through a literature review and analysing the dataset.
This established the problem framing, clarified the type of analytics that would be most useful to the lab instructor during a computer lab and also addressed the limitations of current approaches.

The next stage focused on system design. This included defining the system architecture, understanding the structure of the data endpoint and deciding which data can aid in creating our student struggle and question difficulty metrics.
The process also also considered how the system could be extended to smart devices, ensuring that the design was flexible enough to accommodate future extensions.

Implementation then progressed from core functionality to increasing the complexity of our analytics.
The priority was to establish reliable data retrieval, parsing and processing. 
Once this was in place, we extended the dashboard to include live monitoring, drill down analysis, model comparison, historical sessions and assistant coordination.
More advanced analytical components were implemented such as collaborative filtering, mistake clustering and alternative modelling approaches which use Item Response Theory and Bayesian Knowledge Tracing.

The incremental approach was appropriate for this project as it reduced risk and made it possible to validate each layer of the system before moving on to the next step.
This was approach allowed us to turn a functioning pipeline into a more complete analytics platform.

Overall, the project aligned with the practical aim of the dissertation, which was to produce a working system that improves awareness and prioritisation in live computer labs.
