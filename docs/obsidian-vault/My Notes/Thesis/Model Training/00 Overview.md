# Model Training — overview

This folder is the permanent reference for the v2 training work. Three model-class trainings + one classifier (threshold) training, plus a separate hyperparameter optimisation pass that affects how the underlying scores are computed.

## The three-component empirical-validation arc

The v2 pipeline replaces every hand-set element of the v1 baseline with an empirically-trained alternative, each validated against the GPT-4o 4-band rater under matched cross-validation:

| Component | What's trained | Where |
|---|---|---|
| Composite weights | Per-feature OLS coefficients (3 models) | [[Struggle Model Training]], [[Difficulty Model Training]] |
| Scalar hyperparameters | CF threshold τ, shrinkage K | Optuna TPE, see `scripts/optimise_hyperparams.py` |
| **Score-to-band thresholds** | **3 cutpoints per (model × score-space)** | [[Classifier Training]] |

Component (1) was promoted to canonical 2026-05-26 (relabel pass). Component (2) was promoted same date. Component (3) was promoted 2026-05-27 evening per `[[v2 Threshold Promotion Handoff]]` (now `status: completed`); constrained CV-trained cutpoints now live in `code2/backend/config.py` — struggle (0.000, 0.315, 0.505, 0.605, 1.000); difficulty (0.000, 0.363, 0.463, 0.587, 1.000).

## How struggle / difficulty / classifier relate

```
struggle features ──┐
                    ├──► weighted composite score ──[thresholds]──► band label
difficulty features ┘                                                (instructor sees this)

                     │            │
                     │            │
                 WEIGHTS      CLASSIFIER
                  (OLS)        (κ-max)
                     │            │
                     ▼            ▼
          [[Struggle Model    [[Classifier
            Training]]         Training]]
          [[Difficulty Model
            Training]]
```

Weights training shapes WHO ranks where; classifier training shapes WHAT each rank gets called. Both are cross-validated against the LLM rater. The two pipelines are decoupled (separate fits) for interpretability — joint ordinal regression is flagged as future work.

## File-by-file

- [[Struggle Model Training]] — v2 OLS weights for the 7-signal baseline composite + the 3-signal improved (BKT + IRT) blend. n=1306, GroupKFold(5) by session_id.
- [[Difficulty Model Training]] — v2 OLS weights for the 5-signal composite. n=72, LeaveOneOut.
- [[Classifier Training]] — band-threshold optimisation across all 6 (model × score-space) combinations. Brute-force κ-max grid search, CV-validated, with both unconstrained and constrained variants reported.

## Open questions / future-work seeds

- **Joint weight + cutpoint fit** (proportional-odds logit / `mord` / `statsmodels.OrderedModel`) — could compress the two-stage decoupled fit into one. Trades interpretability of the underlying composite for principled cutpoint estimation.
- **Cohort-balanced re-evaluation** — the trained thresholds and the LLM rater both reflect this cohort's class distribution (0 Easy questions; 5.8% On Track snapshots). A balanced cohort would re-validate the lowest bands.
- **Per-semester threshold re-tuning** — if cohort distributions shift each academic year, the threshold + weight + hyperparam values may need annual re-fit. Deployment-side ML maintenance is an unaddressed concern.
- **IRT rater anchoring** — the IRT-difficulty κ mismatch (see [[Classifier Training]] §3) suggests an LLM rater anchored on response-pattern semantics would test the IRT model on its own terms. Currently we're testing it against a pedagogical-difficulty rater, which measures a different latent quantity.
