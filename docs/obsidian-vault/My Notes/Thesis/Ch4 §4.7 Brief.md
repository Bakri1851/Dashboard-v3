## Corrections to the Ch4 Rewrite Brief (verified against live code)

The base brief had four claims that disagree with the live code. Use the corrections below instead of the originals:

1. **Measurement Confidence uses only two factors, not four.** The live code uses **only feedback length (ramp) and score extremity $|2x-1|$** times a base ceiling. There is no specificity metric, no per-row temperature term, no response variance. See `code/learning_dashboard/models/measurement.py:9-51` and `code2/backend/incorrectness.py` `compute_feedback_confidence`.
2. **`_CONFIDENCE_FLOOR = 0.2` is V2-only.** Defined at `code2/backend/incorrectness.py:356`, used inside `_confidence_weighted_mean()`. V1's `measurement.py` has no equivalent floor.
3. **Confidence has no colour bands.** $\kappa \in [0,1]$ is continuous; the UI renders a gradient. Threshold-band logic is for struggle and difficulty, not confidence.
4. **V2 BKT fits per-skill (module), not per-question.** Documented in `code2/backend/models/bkt.py` docstring. V1's `compute_student_mastery` operates per-question with static defaults; V2's `fit_all_skills` fits one parameter set per module and reuses across questions in that module — a deliberate grain change for pooling power.

Plus three smaller corrections:

- **V2 has no explicit `improved_models_enabled` flag.** V1 uses `st.session_state["improved_models_enabled"]` derived from the Settings dropdowns (`instructor_app.py:880-882`). V2 carries the choice in `runtime_config.RuntimeConfig.struggle_model` and `difficulty_model` as enums (`runtime_config.py:25-26`).
- **Weight redistribution in the improved-struggle aggregator is additive, not proportional.** `improved_struggle.py:71-76` dumps the missing component's weight entirely into $w_B$ (behavioural). Defaults (0.45/0.30/0.25) → BKT off becomes (0.75/—/0.25); IRT off becomes (0.70/0.30/—); both off becomes (1.00/—/—). The brief's earlier proportional-redistribution table is wrong.
- **Config constant names are `IMPROVED_STRUGGLE_WEIGHT_*`, not `IMPROVED_WEIGHT_*`.** Verbatim: `IMPROVED_STRUGGLE_WEIGHT_BEHAVIORAL = 0.45`, `IMPROVED_STRUGGLE_WEIGHT_MASTERY_GAP = 0.30`, `IMPROVED_STRUGGLE_WEIGHT_DIFFICULTY_ADJ = 0.25` (note American "BEHAVIORAL" in the constant name; British "behavioural" in prose).

## §4.7.1 — Measurement Confidence

**Maths recap:** For an incorrectness score $x \in [0,1]$ with feedback text of length $\ell$ characters,

$$\kappa = \mathrm{ramp}_{\ell, 0, L} \cdot (0.5 + 0.5\,|2x-1|) \cdot \kappa_{\text{base}},$$

where $\mathrm{ramp}_{\ell, 0, L}$ rises linearly from 0 to 1 over $[0, L]$ characters ($L = 20$), $|2x-1|$ is extremity (1 at endpoints, 0 at $x = 0.5$), and $\kappa_{\text{base}} = 0.7$.

**Single paragraph (cover):**
- Confidence is a **meta-signal**: how confident are we in the scorer's verdict? Surfaced as a coloured dot in Question Detail (§4.9.3).
- Only two factors: feedback **length** (short/empty collapses $\kappa$ toward 0) and **extremity** (mid-scale verdicts are less trustworthy). Empty feedback returns $\kappa = 0$, not 0.5 — the score is fallback, not measurement.
- V2 adds `_CONFIDENCE_FLOOR = 0.2` in `_confidence_weighted_mean()` so low-confidence rows still contribute to the V2 struggle pipeline. V1 has no equivalent floor.
- $\kappa$ is never banded into discrete colours; the UI renders a continuous gradient (green → amber → grey).

**Voice/length:** Maths recap + 1 paragraph.

**Figure (optional):** 6-line snippet of the confidence formula from `measurement.py`. Parallel to §4.6 snippets. Drop if keeping figure count down.

## §4.7.2 — Item Response Theory (IRT) Difficulty

**Maths recap:** V1 fits Rasch (1PL): $P(X_{s,q} = 1 \mid \theta_s, \beta_q) = \sigma(\theta_s - \beta_q)$. V2 evolves to 2PL: $P(\dots) = \sigma(\alpha_q(\theta_s - \beta_q))$ adding positive per-question discrimination $\alpha_q$. Both fit via joint MLE under L-BFGS-B \cite{byrdLimitedMemoryAlgorithm1995}. 2PL collapses to Rasch when all $\alpha_q = 1$ (V2 is a strict generalisation).

**Paragraph 1 — V1 Rasch:**
- `models/irt.py:67` `fit_rasch_model` builds response matrix (1 if best-attempt incorrectness < 0.5, else 0; NaN if no attempt).
- Filtering: `IRT_MIN_ATTEMPTS_PER_QUESTION = 2`, `IRT_MIN_ATTEMPTS_PER_STUDENT = 2`, applied iteratively.
- L-BFGS-B with `IRT_MAX_ITER = 100`; convergence surfaced as UI status badge.
- $\hat{\beta}_q$ on logit scale; leaderboard maps via $D^{\mathrm{IRT}}(q) = \sigma(\hat{\beta}_q)$. Bands match baseline.

**Paragraph 2 — V2 2PL:**
- `code2/backend/models/irt.py:133` `fit_2pl_model`; orchestrator at line 269; `compute_irt_abilities` at line 348 (V2-only).
- Parameter layout $[\theta, b, \log a]$; discrimination fitted in log-space, exponentiated → positivity for free. Bound `log_a ∈ [-5, +5]` prevents degenerate-discrimination items.
- Identifiability: mean$(\theta) = 0$ and mean$(\log a) = 0$ reapplied each gradient eval.
- Output: `difficulty_df`, `ability_df`, convergence, log-likelihood. Cached in `_irt_difficulty_cache` (TTLCache(maxsize=4, ttl=600), keyed by `(from_iso, to_iso)`).

**Paragraph 3 — analytical commentary:**
- 1PL→2PL is one of two genuine algorithmic V1/V2 differences (other: BKT persistence, §4.7.3).
- $\alpha_q$ near 0 flags items that fail to separate the cohort — free bad-item diagnostic.

**Voice/length:** Maths recap + 3 paragraphs.

**Figure (strong candidate):** 5–6 line snippet of `log_a = ...; a = np.exp(log_a)` + box bound from `fit_2pl_model`. Demonstrates the V1→V2 evolution concretely.

## §4.7.3 — Bayesian Knowledge Tracing (BKT) Mastery

**Maths recap:** Two-state HMM per (student, skill). State $L_t \in \{\text{not learned, learned}\}$, observation $c_t \in \{0,1\}$. Four parameters: $P(L_0) = 0.3$, $P(T) = 0.1$, $P(G) = 0.2$, $P(S) = 0.1$.

Forward update:
$$P(L_t \mid c_t) = \frac{P(L_t) P(c_t \mid L_t)}{P(c_t)}, \quad P(L_{t+1}) = P(L_t \mid c_t) + (1 - P(L_t \mid c_t)) P(T).$$

**Paragraph 1 — V1 static, V2 fitted:**
- V1 uses config defaults (Table `tab:bkt-defaults`); `fit_bkt_parameters` exists at `models/bkt.py:195-337` but is never called. `compute_student_mastery` replays each student's chronological submissions per question to produce final $P(L)$.
- V2 fits once per process via `fit_all_skills` at FastAPI startup (`main.py:141`); caches in `_BKT_PARAMS_CACHE`. Refuses if fewer than `BKT_FIT_MIN_OBSERVATIONS = 50` attempts or single-class data.
- L-BFGS-B with bounds $[0,1]^2 \times [0, 0.5]^2$ — the 0.5 ceilings on $G$ and $S$ implicitly enforce $P(G) + P(S) < 1$.

**Paragraph 2 — V2 per-skill grain + AUC:**
- V2's `fit_all_skills` fits **one parameter set per skill (module), not per question** — pools across questions in a subject area where per-question fits would be data-starved.
- Predictive AUC via `sklearn.metrics.roc_auc_score` reported alongside the fit; the headline diagnostic for Ch5's baseline-vs-BKT comparison.
- `BKT_MASTERY_THRESHOLD = 0.95` is the bar for the "mastered" badge on Student Detail (§4.9.2).

**Drop-in LaTeX table** (`tab:bkt-defaults`): 4 rows for $P(L_0), P(T), P(G), P(S)$ with default and fit bounds. Full block in the plan file.

**Voice/length:** Maths recap + 2 paragraphs + 1 table.

**Citations:** `\cite{corbettKnowledgeTracingModeling1995}` (already in Ch3); reuse `\cite{byrdLimitedMemoryAlgorithm1995}` from §4.7.2.

## §4.7.4 — Improved Struggle Model

**Maths recap:**
$$S_{\text{improved}}(s) = w_B B_s + w_M M_s + w_D A_s,$$
with $B_s$ behavioural composite (§4.6.2), $M_s = 1 - \overline{P(L)}_s$ mastery gap (§4.7.3), $A_s$ difficulty-adjusted exposure. Default $(w_B, w_M, w_D) = (0.45, 0.30, 0.25)$.

**Paragraph 1 — V1 weighted composite:**
- `models/improved_struggle.py:24-195`; weighted sum at lines 165–176.
- V1's difficulty-adjustment: `incorrectness × (1 - norm_diff)` in `_compute_difficulty_adjusted` (lines 202–261).
- **Graceful weight redistribution**: missing BKT absorbs $w_M$ entirely into $w_B$ ($w_D$ unchanged); missing IRT absorbs $w_D$ entirely into $w_B$. The score is always defined.

**Paragraph 2 — V2 IRT-residual path:**
- V2's entry takes `irt_ability` kwarg. When full 2PL params present, `_compute_irt_residual` at lines 301–378 replaces the legacy term with expected $\sigma(\alpha_q(\theta_s - \beta_q))$ minus observed correctness, summed across submissions, rescaled to $[0,1]$.
- Intuition: a student tracking below their IRT-expected performance is struggling more than a student tracking expectation, regardless of overall difficulty exposure. Falls back to V1 legacy when 2PL params absent.
- Toggle plumbing: V1 — `st.session_state["improved_models_enabled"]` derived from Settings dropdowns; V2 — `runtime_config.struggle_model` and `difficulty_model` enums, persisted via `POST /api/settings`.

**Table** — weight-redistribution cases (additive: missing weight dumps entirely into $w_B$):

| Available | $w_B$ | $w_M$ | $w_D$ |
|---|---|---|---|
| All three | 0.45 | 0.30 | 0.25 |
| BKT off | 0.75 | — | 0.25 |
| IRT off | 0.70 | 0.30 | — |
| Both off | 1.00 | — | — |
