# Zotero ↔ Obsidian Workflow

How citations flow from Zotero into Obsidian literature notes and into the LaTeX thesis. Update this note when the workflow changes.

Related: [[Setup and Runbook]] · [[Literature/index|Literature Index]] · [[Literature/coverage|Coverage Index]] · [[Report Sync]]

---

## Stack

- **Zotero desktop** with **Better BibTeX** plugin → owns the bibliographic database; auto-exports `Report/references.bib` whenever an item is added or edited
- **Obsidian Zotero Integration plugin** (`mgmeyers/obsidian-zotero-integration`) → pulls metadata, abstracts, and PDF annotations from Zotero into Obsidian notes
- **`scripts/sync_literature.py`** → keeps each lit note's frontmatter, the `Cited in:` line, and `Literature/coverage.md` in sync with the actual `\cite{}` calls in `Report/**/*.tex`

The thesis itself uses standard `\cite{}` against `references.bib` — the Obsidian layer is for note-taking and coverage tracking only.

---

## One-time setup

### 1. Better BibTeX (Zotero side)

In Zotero → `Edit → Settings → Better BibTeX`:

- **Citation key formula:** `auth.lower + "_" + year + "_" + title.lower.select(1,1)` (matches the existing pattern `surname_year_firstword`, e.g. `ahn_2019_the`)
- **Automatic export:** select the library/collection that owns thesis citations → right-click → `Export Library` → format `Better BibTeX`, **tick "Keep updated"**, save as `Report/references.bib` in this repo
- Verify: edit any field on a Zotero item, save, then `git status` should show `Report/references.bib` modified within ~1 s

### 2. Zotero connector port

In Zotero → `Edit → Settings → Advanced → General → Config Editor`:

- Search `extensions.zotero.httpServer.enabled` → must be `true`
- Default port is `23119`. The Obsidian plugin uses this; nothing to change unless you've moved it.

### 3. Obsidian — install Dataview (optional)

`Literature/coverage.md` is a static markdown file generated by `sync_literature.py`, so Dataview is **not required**. Install it only if you want richer live queries on top of the lit-note frontmatter (e.g. filtering by `status: planned` while writing).

### 4. Obsidian Zotero Integration — configure

`Settings → Community Plugins → Zotero Integration` (already enabled, just unconfigured).

- **Database:** `Zotero` (not Juris-M)
- **Citation format** (one entry, name = "Pandoc cite"):
    - Format: `Pandoc`
    - Brackets: `[…]`
    - Command name: `Insert Pandoc citation`
- **Import format** (one entry, name = "Lit Note"):
    - Output path: `My Notes/Literature/{{citekey}}.md` *(no `@` prefix — matches existing 88 notes)*
    - Image output path: leave blank (no PDF images yet)
    - Template path: `My Notes/_templates/zotero-literature.md`
    - Tick `Open created/updated note after import` if you want to jump to it
- **Hotkeys** (optional but useful): bind `Ctrl+Shift+Z` to `Zotero Integration: Lit Note`

You should now see two new commands in the command palette:

- `Zotero Integration: Lit Note` — picks a Zotero item, renders the template into `Literature/<citekey>.md`. **Re-running on the same paper** preserves whatever is inside `%% Begin annotations %% … %% End annotations %%` and appends new PDF annotations.
- `Zotero Integration: Insert Pandoc citation` — inserts `[@citekey]` at the cursor (for in-vault prose; thesis still uses `\cite{}`)

## Day-to-day workflow

### Adding a new paper

1. **Zotero:** drag PDF in (or use browser connector). Better BibTeX assigns a citekey and updates `Report/references.bib` automatically.
2. **Annotate the PDF inside Zotero** with colour-coded highlights:
    - **Yellow** — key idea / notable claim
    - **Red** — methodological concern / critique
    - **Green** — directly supports a thesis claim, candidate for `\cite{}`
    - **Blue** — useful definition / quote
   Use the Note tool for free-text notes attached to a highlight. The template renders highlights as `> [!Quote]` callouts grouped by colour.
3. **Obsidian:** command palette → `Zotero Integration: Lit Note` → pick the new paper. A new note `Literature/<citekey>.md` appears with the template's structure. The `## Summary` heading inside the annotations block is your hand-written contribution — fill it in.
4. **`\cite{}` in LaTeX:** insert `\cite{<citekey>}` wherever needed in `Report/**/*.tex`.
5. **Sync:** run `python scripts/sync_literature.py` from repo root. The new note's frontmatter gets `status: active`, `cited_in_tex` populated, and `coverage.md` is regenerated.

### Re-importing after more annotations

You highlight more of the PDF in Zotero later. Run `Zotero Integration: Lit Note` again on the same paper:

- The plugin reads the existing note's `%% Begin annotations %% … %% End annotations %%` block
- Renders a fresh template, but the new render's annotations section contains only **new** highlights (filtered by `lastImportDate`)
- Final file = template with the **existing block content + new highlights appended below**
- Your `## Summary` content survives because it lives inside the annotations block

The frontmatter gets rewritten by the template too — that's fine, `sync_literature.py` re-augments it after.

### Promoting a "planned" citation to "active"

A planned note has a hand-written stub but the paper isn't in Zotero yet (or isn't `\cite{}`d yet). To promote:

1. Add the paper to Zotero → Better BibTeX writes the bib entry. The citekey **must match** the existing note's filename, otherwise you get a duplicate. Tweak Better BibTeX's "Pin citekey" if needed.
2. In Obsidian, run `Zotero Integration: Lit Note` on it. The plugin will overwrite the file with the template — but the existing 3-line summary has already been wrapped in `%% Begin annotations %%` markers by `sync_literature.py`, so it survives.
3. Insert `\cite{}` calls in the relevant `.tex` files.
4. Run `python scripts/sync_literature.py`. `status` flips to `active` in the frontmatter and on `coverage.md`.

### Adding a new "planned" citation by hand

If a paper isn't ready to go into Zotero but you want to flag it:

1. Create `Literature/<citekey>.md` manually with the existing 3-line format:

       # Author — Title
       
       1-line context note.
       
       Cited in: [[Ch3 – Design and Modelling]] · [[Some Method Note]]

2. Run `python scripts/sync_literature.py`. Frontmatter is added; the note appears under "Planned" in `coverage.md`.

## Files

| Path | Owner | Purpose |
|---|---|---|
| `Report/references.bib` | Better BibTeX (Zotero) | Source of truth for available citekeys |
| `Report/**/*.tex` | You (LaTeX) | `\cite{}` calls — the actual citations in the thesis |
| `docs/obsidian-vault/My Notes/Literature/<citekey>.md` | You + Zotero Integration plugin + sync script | Per-paper notes; hand-summary inside annotation block, frontmatter machine-managed |
| `docs/obsidian-vault/My Notes/Literature/index.md` | You | Curated thematic groupings of papers (hand-edited) |
| `docs/obsidian-vault/My Notes/Literature/coverage.md` | sync script | Auto-generated active/planned/stale matrix and chapter index |
| `docs/obsidian-vault/My Notes/_templates/zotero-literature.md` | You | Nunjucks template for Zotero Integration imports |
| `scripts/sync_literature.py` | You | Frontmatter + coverage regenerator |

## Troubleshooting

**"Could not connect to Zotero" in Obsidian.** Zotero desktop must be running with the HTTP server enabled. Check `extensions.zotero.httpServer.enabled` in Zotero's config editor.

**Template renders nothing / blank file.** Template path in plugin settings must match the actual file. Default path is relative to the vault root, e.g. `My Notes/_templates/zotero-literature.md`.

**Hand-written summary disappears after re-import.** The summary must be **inside** `%% Begin annotations %%` … `%% End annotations %%`. If `sync_literature.py` hasn't been run on a note yet, run it first — the script wraps existing summaries automatically for `status: active` notes.

**Citekey mismatch between Zotero and existing note.** Use Better BibTeX's "Pin citation key" on the Zotero item (right-click → Better BibTeX → Pin Citation Key) to force the key the existing note expects.

**Coverage page is stale.** Run `python scripts/sync_literature.py`. It's idempotent and safe to re-run anytime.
